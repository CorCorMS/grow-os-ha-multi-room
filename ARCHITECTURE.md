# Grow OS HA Multi-Room Architecture

## Goal

Grow OS HA Multi-Room is intended to extend the single-room Arbiter-first
approach toward several rooms, cabinets, or zones.

## Planned principles

- one clear policy source per room
- shared conventions across rooms
- isolated room-level execution
- reusable multi-room dashboard patterns
- migration path from single-room Grow OS HA

## Planned layers

- shared base concepts
- room-specific registers
- room-specific arbiters
- shared installer logic
- overview dashboards across rooms
